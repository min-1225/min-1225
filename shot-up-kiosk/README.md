# Shot Up 카페 키오스크

터치스크린 카페 키오스크 UI 포트폴리오 프로젝트입니다.

매장/포장 선택, 한영 언어 전환, 카테고리별 메뉴 주문, 장바구니 관리, 커피류 샷 추가, 결제 방법 선택, 주문 완료까지 실제 키오스크와 유사한 전체 주문 플로우를 구현했습니다.

## Demo

https://kiosk-bice-ten.vercel.app/

## 주요 기능

| 기능 | 설명 |
| --- | --- |
| 홈 화면 | 매장 / 포장 선택 후 주문 시작 |
| 한영 전환 | 홈 화면에서 언어 선택 후 전체 UI 즉시 전환 |
| 메뉴 이미지 | 카테고리별 메뉴 이미지 표시 |
| 카테고리 사이드바 | 커피류 / 음료류 / 차류 / 디저트류 메뉴 탐색 |
| 장바구니 | 수량 조절, 삭제, 전체 취소 |
| 샷 추가 | 커피류 한정, 최대 2샷, 샷당 +500원 |
| 결제 | 신용카드 / 삼성페이 결제 방법 선택 |
| 주문 완료 | 주문 번호 표시, 5초 후 홈 화면 자동 복귀 |

## 화면 플로우

```text
홈 (매장/포장 + 언어 선택)
  └─▶ 메뉴 (카테고리 사이드바 + 메뉴 그리드 + 장바구니)
        └─▶ 주문 확인
              └─▶ 결제 방법 선택
                    └─▶ 주문 완료
                          └─▶ 홈 자동 복귀
```

## 메뉴 카테고리

| 카테고리 | 메뉴 수 | 기본 가격 |
| --- | ---: | ---: |
| 커피류 | 6종 | 2,000원 |
| 음료류 | 5종 | 4,000원 |
| 차류 | 5종 | 2,000원 |
| 디저트류 | 4종 | 4,000원 |

## 샷 추가 로직

커피류 메뉴에만 샷 추가 옵션을 표시합니다.

- 최대 2샷까지 추가 가능
- 샷당 500원 추가
- 커피류가 아닌 메뉴에는 샷 옵션 미표시
- 장바구니 금액에 샷 추가 금액 즉시 반영

## 구현 포인트

- 터치스크린 환경을 고려한 큰 버튼과 명확한 상태 표시
- 메뉴 선택부터 결제 완료까지 끊기지 않는 주문 플로우
- 장바구니 인라인 수량 조절
- 커피류에만 조건부로 노출되는 샷 추가 UI
- 주문 완료 후 5초 자동 복귀 처리
- 한국어/영어 UI 전환

## 프로젝트 목적

이 프로젝트는 단순 정적 페이지가 아니라, 실제 키오스크에서 사용자가 거치는 주문 흐름을 프론트엔드 상태 관리로 구현하는 것을 목표로 했습니다.

포트폴리오 관점에서는 다음 역량을 보여주기 위한 프로젝트입니다.

- 사용자 플로우 설계
- 조건부 UI 렌더링
- 장바구니 상태 관리
- 가격 계산 로직
- 결제 단계 화면 전환
- 배포 가능한 웹앱 제작
